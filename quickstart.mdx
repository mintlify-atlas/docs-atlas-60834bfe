## Prerequisites

- Python 3.7 or later
- `pip` package manager

<Note>
  `tkinter` is included with most Python distributions. On some Linux systems you may need to install it separately — see the [FAQ](/faq) for details.
</Note>

## Installation

<Steps>
  <Step title="Clone the repository">
    ```bash
    git clone https://github.com/EdwinHernanGilRodriguez/data_analyzer.git
    cd data_analyzer
    ```
  </Step>
  <Step title="Create a virtual environment (recommended)">
    <Tip>
      Using a virtual environment keeps your dependencies isolated from other Python projects.
    </Tip>
    <CodeGroup>
      ```bash macOS / Linux
      python3 -m venv venv
      source venv/bin/activate
      ```
      ```bash Windows
      python -m venv venv
      venv\Scripts\activate
      ```
    </CodeGroup>
  </Step>
  <Step title="Install dependencies">
    The application requires `pandas` for data processing and `tabulate` for formatting results.

    ```bash
    pip install pandas tabulate
    ```
  </Step>
  <Step title="Launch the application">
    ```bash
    python src/gui.py
    ```

    The Data Analyzer window opens with the title **"Data Analyzer - By Edwin Gil"**.
  </Step>
</Steps>

## Using the GUI

<Steps>
  <Step title="Load a CSV file">
    Click the **Cargar CSV** button. A file dialog opens filtered to `.csv` files. Select your file and click **Open**.

    The first 5 rows of your data appear in the text pane as a preview.
  </Step>
  <Step title="Choose analysis type">
    An **Opciones de Análisis** popup appears with two options:

    - **Analizar todo el archivo** — analyze every column in the dataset
    - **Seleccionar columnas** — choose specific columns to analyze
  </Step>
  <Step title="Review the results">
    The statistical summary is displayed in the scrollable text area as a formatted grid table. Scroll down to see all columns.
  </Step>
</Steps>

## Programmatic usage

You can also use the analysis functions directly in Python without the GUI.

<Note>
  The import paths below assume your script runs from within the `src/` directory, or that `src/` is on your Python path. This matches how `gui.py` imports from `analyzer`.
</Note>

### Analyze a CSV file from a path

```python example_usage.py
from analyzer import analizar_csv

result = analizar_csv("data/sales.csv")
if result is not None:
    print(result)
else:
    print("Failed to analyze file — check the path and format.")
```

### Analyze a pandas DataFrame directly

```python
import pandas as pd
from analyzer import generar_resumen_estadistico

df = pd.read_csv("data/sales.csv")
summary = generar_resumen_estadistico(df)
print(summary)
```

### Analyze specific columns

```python
import pandas as pd
from analyzer import generar_resumen_estadistico

df = pd.read_csv("data/sales.csv")
# Select only the columns you care about
df_subset = df[["price", "quantity", "category"]]
summary = generar_resumen_estadistico(df_subset)
print(summary)
```

### Expected output

The output is a grid-formatted table. Here's an example for a CSV with numeric and text columns:

```
+----------+-------+--------+---------+-----+------+------+------+-----+
|          | count |  mean  |   std   | min | 25%  | 50%  | 75%  | max |
+----------+-------+--------+---------+-----+------+------+------+-----+
| price    | 100.0 | 45.23  | 12.84   | 5.0 | 35.0 | 45.0 | 55.0 | 99  |
| quantity |  100  |   3.5  |   1.2   | 1.0 |  2.0 |  3.0 |  5.0 | 10  |
+----------+-------+--------+---------+-----+------+------+------+-----+
```

<Note>
  Categorical (text) columns show `count`, `unique`, `top`, and `freq` instead of numeric statistics.
</Note>
